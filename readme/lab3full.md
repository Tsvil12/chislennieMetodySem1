# Численное интегрирование функции cos(5x²)

## Описание
Вычисление интеграла ∫₀⁵ cos(5x²) dx с точностью ε=10⁻⁶

## Файлы и методы

### lab3n11.py — Метод трапеций
- **Оценка погрешности**: R = (b-a)/12 * M₂ * h²
- **Адаптация**: увеличение n до выполнения |R| ≤ ε/2
- **Строки**: 14-19 (адаптация), 18-19 (формула)

### lab3n12.py — Метод Симпсона
- **Оценка погрешности**: R = -(b-a)/180 * M₄ * h⁴
- **Адаптация**: удвоение n до выполнения |R| ≤ ε/2
- **Строки**: 25-30 (адаптация), 32 (формула)

### lab3n13.py — Метод левых прямоугольников
- **Оценка погрешности**: R = -(b-a)³/(24n²) * M₂
- **Адаптация**: увеличение n на 1
- **Строки**: 17-26 (адаптация), 28-32 (формула)

### lab3n14.py — Метод Гаусса (n=4)
- **Узлы и веса**: фиксированные для n=4
- **Погрешность**: зависит от f⁽⁸⁾(x)
- **Строки**: 14-15 (узлы/веса), 21 (интеграл)

### lab3n2.py — Сравнение способов выбора шага
- **Способ 1**: по оценке остаточного члена (строки 48-63)
- **Способ 2**: последовательное удвоение шагов (строки 65-86)
- **Методы**: трапеции и Симпсона

## Вывод
Метод Гаусса — самый точный при малом числе вычислений.  
Метод удвоения шагов — универсальный и надёжный.









Общая задача всех программ
Все программы решают одну и ту же математическую задачу — численное вычисление определённого интегралас заданной точностью ε=10^−6


Что происходит в коде в целом?
1. Основная идея численного интегрирования
Любой численный метод заменяет интеграл суммой площадей простых геометрических фигур:

Прямоугольники — площадь под графиком заменяется суммой прямоугольников

Трапеции — суммой трапеций

Симпсон — суммой параболических сегментов

Гаусс — взвешенной суммой значений в специальных точках

Все методы следуют общей схеме:

Разбить интервал на n маленьких отрезков

На каждом отрезке приблизить функцию простой формой

Сложить площади

Оценить погрешность

2. Два подхода к достижению точности
В программах реализованы два способа обеспечить точность 

Способ 1: Оценка остаточного члена (теоретический)
Вычисляется максимальное значение производной функции на интервале

По формуле погрешности определяется необходимое количество отрезков 

Интеграл вычисляется один раз с найденным n

Реализовано в: lab3n11.py, lab3n12.py, lab3n13.py
